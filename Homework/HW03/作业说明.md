## 01 你的任务

要求：在至多 100（含 100）个 epoch 之内，使测试集的准确率达到 80% 以上，并记录每个 epoch 结束时的训练集和测试集准确率。（1 分）

## 03 一些阅读材料

### 3.1 关于 MLU

```
use_mlu = False
try:
    use_mlu = torch.mlu.is_available()
except:
    use_mlu = False

if use_mlu:
    device = torch.device('mlu:0')
else:
    print("MLU is not available, use GPU/CPU instead.")
    if torch.cuda.is_available():
        device = torch.device('cuda:0')
    else:
        device = torch.device('cpu')

model = model.to(device)
```

MLU 是寒武纪公司推出的专用AI 加速器，主要用于深度学习推理和训练任务。我们可以简单地将其理解为“国产GPU”。
需要说明的是：要使用 MLU，仅仅在代码中写`torch.device('mlu')` 是不够的。它需要先安装寒武纪为 PyTorch 提供的插件（通常叫 `torch_mlu` 或 `Cambricon PyTorch`），就像在常规的 NVIDIA GPU 上需要安装 torch 的 CUDA 版本一样。
大多数同学是不具有该加速器的，但上述代码提供了异常处理 `try - except` ，所以并不会影响代码的运行。有多个显卡的同学可能需要根据自己的设备情况修改上述代码中的设备选择逻辑以保证选择正确的设备，或者直接删除这些内容，自己写一个设备选择逻辑，或直接硬编码指定。

### 3.2 关于重定向输出

在训练的过程中，我们需要输出训练集和测试集的准确率等信息。通常我们会使用print 函数来输出这些信息，这样会把所有的日志直接输出到终端上。复制并不方便，也不利于后续分析。除了修改Python 代码，使用文件写入的方式来记录这些信息之外，我们还可以使用重定向输出的方式来将这些信息保存到一个日志文件中。

你需要把你运行脚本的指令改成：
```
python cifar10_cnn.py > training.log 2>&1
```

这里的 `>` 是重定向输出的符号，表示将标准输出 `stdout` 重定向到 `training.log` 文件中。`2>&1` 表示将标准错误 `stderr`（设备编号 2）也重定向到标准输出 `stdout`（设备编号 1）中，这样就可以将所有的输出都保存到该文件中。

如果要追加输出到文件中而不是覆盖，可以使用`>>` 符号：

```
python cifar10_cnn.py >> training.log 2>&1
```
