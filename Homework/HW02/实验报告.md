# 实验报告

## 作业1

### Version 1

补充代码内容、将损失函数修改为交叉熵损失函数，得到单个 hidden layer 效果：

![](result\np1.png)

**分析**：交叉熵损失函数更适合多分类任务，相比于原始的均方误差，交叉熵加快了收敛速度。

### Version 2

调整网络结构，使用两个 hidden layer，分别 256 个神经元：

![](result\np2.png)

**分析**：将网络结构加深为两层隐藏层，提高了模型对特征的非线性表达能力和参数容量，最终测试集准确率稳定在95%以上。

### Version 3

修改激活函数，分别使用 ReLu 和 tanh（第一层 ReLU，第二层 Tanh）：

![](result\np3.png)

**分析**：意外发现混合激活函数的效果极佳。第一层采用 ReLU 提取稀疏化特征且保证梯度顺利下传，第二层采用 tanh 将特征数据零中心化并平滑映射到 (-1,1) 区间，这种特性的互补使得最后一层分类边界更加稳定。

### Version 4

都使用 tanh：

![](result\np4.png)

**分析**：效果不如纯 ReLU。因为 tanh 在输入偏大时会进入两侧饱和区，导数趋于 0。连续多层使用会导致反向传播时的梯度消失，初期收敛非常缓慢。

### Version 5

都使用 Sigmoid：

![](result\np5.png)

**分析**：表现最差且收敛极慢。Sigmoid 的导数最大仅为 0.25，连乘会引发极其严重的梯度消失；且其输出恒为正数（非零中心化），导致权重更新方向受限，网络前半部分几乎无法有效学习。

*最终采用 Version 2*

## 作业2

### Version 0

运行 template 代码，初始状态结果：

![](result\torch0.png)

![](result\Figure_0.png)

**分析**：采用 800 -> 800 -> 10 的两层隐藏层结构，即便随机初始化没有额外技巧，因为隐藏单元数量多，模型也能较快收敛。

### Version 1

增加隐藏层，调整神经元数量：

![](result\torch1.png)

![](result\Figure_1.png)

**分析**：将模型调整为更深的 512 -> 256 -> 128 -> 10 结构后，整体效果有了进一步的提升。对比原本极为宽浅的结构，逐层降维的深层网络迫使模型在每层向前传递时不断浓缩信息，提取出更加高级、抽象的非线性特征。这种多层级的特征提取机制极大地增强了模型的表达能力和特征整合能力，使其在验证集上的表现变得更优。

### Version 2

使用 Adam 优化器：

![](result\torch2.png)

![](result\Figure_2.png)

**分析**：将默认的 SGD 替换为 Adam 优化器后，训练速度获得了极其显著的提升。Adam 自带的自适应动量机制有效克服了深层网络梯度微弱与局部鞍点问题，仅需几个 epoch 测试集准确率便轻松突破 97%。同时，可以观察到在最后几个 epoch 中 train accuracy 趋近 100%，而 validation accuracy 无上升趋势，表现出轻微的过拟合现象。

### Version 3

添加 Dropout 层：

![](result\torch3.png)

![](result\Figure_3.png)

**分析**：在每层隐藏层激后添加 Dropout(0.2) 后，通过让神经元随机失活，迫使全连接网络避免对局部特征的严重同质化依赖。虽然牺牲了少量虚高的 train accuracy，但有效缩小了泛化差距，减少了过拟合。

*最终采用 Version 3*
